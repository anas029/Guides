1
Step 1: Check Python and Pip Installation

First, make sure you have Python and Pip installed on your system.

Windows

  • python --version
  • pip --version

Mac

  • python3 --version
  • pip3 --version
2
Step 2: Install Django, Psycopg2, and Django Rest Framework

Install the required Python packages for your Django Rest Framework app.

Windows

  • pip install django psycopg2 djangorestframework

Mac

  • pip3 install django psycopg2 djangorestframework
3
Step 3: Create a PostgreSQL Database

Create a PostgreSQL database for your Django project.

psql

  • CREATE DATABASE my_database_name;
4
Step 4: Change to Your Desired Directory

Open a terminal or command prompt and change to your desired directory.

Terminal

  • cd /path/to/your/directory
5
Step 5: Start a New Django Project

Create a new Django project with your desired project name.

Windows

  • django-admin startproject my_project

Mac

  • django-admin startproject my_project
6
Step 6: Run the Django Development Server

Start the Django development server to test your project. Do not stop the server

Windows

  • cd my_project
  • python manage.py runserver

Mac

  • cd my_project
  • python3 manage.py runserver
7
Step 7: Start a New Django App

In a new terminal

Open the project in VS Code

Terminal

  • code .

Create a new Django app inside your project.

Windows

  • python manage.py startapp my_app

Mac

  • python3 manage.py startapp my_app
8
Step 8: Add Installed Apps in settings.py

In my_project/settings.py file, add the newly created app and the 'rest_framework' app to the 'INSTALLED_APPS' list.

settings.py

  • 'my_app',
  • 'rest_framework',

Your INSTALLED_APPS should looks similar as this:

  • INSTALLED_APPS = [
  •   'django.contrib.admin',
  •   'django.contrib.auth',
  •   'django.contrib.contenttypes',
  •   'django.contrib.sessions',
  •   'django.contrib.messages',
  •   'django.contrib.staticfiles',
  •   'my_app',
  •   'rest_framework',
  • ]
9
Step 9: Add Database Configuration in settings.py

Configure the database settings in the settings.py file to connect Django to your local PostgreSQL database.

settings.py

  • DATABASES = {
  • 'default': {
  • 'ENGINE': 'django.db.backends.postgresql',
  • 'NAME': 'my_database_name',
  • 'USER':'postgres',
  • 'PASSWORD':'my_password',
  • }
  • }
10
Step 10: Create the Book Model

Create the Book model in the models.py file of the app.

models.py

  • class Book(models.Model):
  •     title = models.CharField(max_length=200)
  •     publication_date = models.DateField()
  •     isbn = models.CharField(max_length=13, unique=True)
  •     genre = models.CharField(max_length=100)
  •     description = models.TextField()
  •     price = models.DecimalField(max_digits=6, decimal_places=2)
  •     def __str__(self):
  •         return self.title
11
Step 11: Make Migrations and Apply Changes to the Database

Create and apply migrations to sync the database schema with the Django models.

Windows

  • python manage.py makemigrations
  • python manage.py migrate

Mac

  • python3 manage.py makemigrations
  • python3 manage.py migrate
12
Step 12: Register the Book Model in the Admin Panel

Register the Book model in the admin.py file of the app.

admin.py

  • from .models import Book
  • # Register your models here.
  • admin.site.register(Book)
13
Step 13: Create a Superuser

Create a superuser account to access the Django admin panel.

Windows

  • python manage.py createsuperuser

Mac

  • python3 manage.py createsuperuser
14
Step 14: Create Serializer for the Book Model

Create a serializer for the Book model in a new file called serializer.py inside the my_app folder.

my_app/serializers.py

  • from rest_framework import serializers
  • from .models import Book
  • class BookSerializer(serializers.ModelSerializer):
  •     class Meta:
  •         model = Book
  •         fields = '__all__'
15
Step 15: Create List and Create API Views

Create API views for listing and creating books in the views.py file of the app.

views.py

  • from rest_framework.generics import ListCreateAPIView
  • from .models import Book
  • from .serializers import BookSerializer
  • # Create your views here.
  • class BookListCreate(ListCreateAPIView):
  •     queryset = Book.objects.all()
  •     serializer_class = BookSerializer
16
Step 16: Create urls.py File

Create a urls.py file inside the my_app folder to define the URLs for the API views.

my_app/urls.py

  • from django.urls import path
  • from .views import BookListCreate
  • urlpatterns = [path('api/books/', BookListCreate.as_view())]
17
Step 17: Include App URLs in Project URLs

Add the URLs of the my_app app to the my_project/urls.py file to include them in the project URLs.

my_project/urls.py

  • from django.contrib import admin
  • from django.urls import path, include
  • urlpatterns = [
  • path('admin/', admin.site.urls),
  • path('', include('my_app.urls')),
  • ]

Chrome

Step 19: Implement Retrieve, Update, and Delete API Views

Add API views for retrieving, updating, and deleting book instances in the views.py file of the app.

views.py

  • from rest_framework.generics import ListCreateAPIView, RetrieveAPIView, UpdateAPIView, DestroyAPIView
  • class BookListRetrieve(RetrieveAPIView):
  •     queryset = Book.objects.all()
  •     serializer_class = BookSerializer
  • class BookListUpdate(UpdateAPIView):
  •     queryset = Book.objects.all()
  •     serializer_class = BookSerializer
  • class BookListDelete(DestroyAPIView):
  •     queryset = Book.objects.all()
  •     serializer_class = BookSerializer
20
Step 20: Add URLs for Retrieve, Update, and Delete API Views

Define the URLs for the retrieve, update, and delete API views in the urls.py file of the app.

my_app/urls.py

  • from django.urls import path
  • from .views import BookListCreate, BookListRetrieve, BookListUpdate, BookListDelete
  • urlpatterns = [
  • path('api/books/', BookListCreate.as_view()),
  • path('api/books/<pk>/retrieve/', BookListRetrieve.as_view()),
  • path('api/books/<pk>/update/', BookListUpdate.as_view()),
  • path('api/books/<pk>/delete/', BookListDelete.as_view()),
  • ]
21

Open your web browser and navigate to the URLs for the retrieve, update, and delete API views to test them.

Chrome

  • http://localhost:8000/books/<book_id>/update/
  • http://localhost:8000/books/<book_id>/delete/
