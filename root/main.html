Step-by-Step Guide: Django Rest Framework

1
Step 1: Check Python and Pip Installation

First, make sure you have Python and Pip installed on your system.

Windows

  • python --version
  • pip --version

Mac

  • python3 --version
  • pip3 --version
2
Step 2: Install Django, Psycopg2, and Django Rest Framework

Install the required Python packages for your Django Rest Framework app.

Windows

  • pip install django
  • pip install psycopg2
  • pip install djangorestframework

Mac

  • pip3 install django
  • pip3 install psycopg2
  • pip3 install djangorestframework
3
Step 3: Create a PostgreSQL Database

Create a PostgreSQL database for your Django project.

psql

  • CREATE DATABASE my_database_name;
4
Step 4: Change to Your Desired Directory

Open a terminal or command prompt and change to your desired directory.

Terminal

  • cd /path/to/your/directory
5
Step 5: Start a New Django Project

Create a new Django project with your desired project name.

Windows

  • django-admin startproject my_project

Mac

  • django-admin startproject my_project
6
Step 6: Run the Django Development Server

Start the Django development server to test your project.

Windows

  • cd my_project
  • python manage.py runserver

Mac

  • cd my_project
  • python3 manage.py runserver
7
Step 7: Start a New Django App

In a new terminal

Open the project in VS Code

Terminal

  • code .

Create a new Django app inside your project.

Windows

  • python manage.py startapp my_app

Mac

  • python3 manage.py startapp my_app
8
Step 8: Add Installed Apps in settings.py

In my_project/settings.py file, add the newly created app and the 'rest_framework' app to

the 'INSTALLED_APPS' list.

VS Code

  • 'my_app',
  • 'rest_framework',

Your INSTALLED_APPS should looks similar as this:

  • INSTALLED_APPS = [
  •   'django.contrib.admin',
  •   'django.contrib.auth',
  •   'django.contrib.contenttypes',
  •   'django.contrib.sessions',
  •   'django.contrib.messages',
  •   'django.contrib.staticfiles',
  •   'my_app',
  •   'rest_framework',
  • ]
9
Step 9: Add Database Configuration in settings.py

Configure the database settings in the settings.py file to connect Django to your local PostgreSQL database.

VS Code

  • DATABASES = {
  • 'default': {
  • 'ENGINE': 'django.db.backends.postgresql',
  • 'NAME': 'cheesedb',
  • 'USER':'postgres',
  • 'PASSWORD':'my_password',
  • }
  • }
10
Step 10: Create the Book Model

Create the Book model in the models.py file of the app.

Windows

  • notepad my_app/models.py

Mac

  • nano my_app/models.py
11
Step 11: Make Migrations and Apply Changes to the Database

Create and apply migrations to sync the database schema with the Django models.

Windows

  • python manage.py makemigrations
  • python manage.py migrate

Mac

  • python3 manage.py makemigrations
  • python3 manage.py migrate
12
Step 12: Register the Book Model in the Admin Panel

Register the Book model in the admin.py file of the app.

Windows

  • notepad my_app/admin.py

Mac

  • nano my_app/admin.py
13
Step 13: Run the Development Server

Start the Django development server to test your project with the newly added Book model and admin panel configuration.

Windows

  • python manage.py runserver

Mac

  • python3 manage.py runserver
14
Step 14: Create Serializer for the Book Model

Create a serializer for the Book model in a new file called serializer.py inside the my_app folder.

Windows

  • notepad my_app/serializer.py

Mac

  • nano my_app/serializer.py
15
Step 15: Create List and Create API Views

Create API views for listing and creating books in the views.py file of the app.

Windows

  • notepad my_app/views.py

Mac

  • nano my_app/views.py
16
Step 16: Create urls.py File

Create a urls.py file inside the my_app folder to define the URLs for the API views.

Windows

  • notepad my_app/urls.py

Mac

  • nano my_app/urls.py
17
Step 17: Include App URLs in Project URLs

Add the URLs of the my_app app to the my_project/urls.py file to include them in the project URLs.

Windows

  • notepad my_project/urls.py

Mac

  • nano my_project/urls.py

Step 29: Test the Relationships in the Django Shell

Open the Django shell and test the relationships by retrieving book objects and accessing their related publisher and author attributes.

Windows

  • python manage.py shell
  • ...
  • >>> from my_app.models import Book
  • >>> book = Book.objects.get(id=1)
  • >>> book.publisher
  • >>> book.author
  • ...
  • >>> exit()

Mac

  • python manage.py shell
  • ...
  • >>> from my_app.models import Book
  • >>> book = Book.objects.get(id=1)
  • >>> book.publisher
  • >>> book.author
  • ...
  • >>> exit()
